Background

Stub-O-Matic is a service which provides data stubs for software testing. Stubs enable the software under test to be isolated from 3rd party systems enabling repeated, automated testing.

Stubo is capable of supporting functional, load and performance testing. Stubbing enables development to progress when the actual dependencies are not available. Stub-O-Matic includes just enough intelligence to emulate back-end systems without the complexity often associated with data set-up activities.

The resilience of automated testing is improved because it is possible to manage stubs more easily than 3rd party systems and databases. Dependant systems are often not capable of creating the full range of test conditions necessary to fully test code. 

Applicability

Use Stub-O-Matic when you don't have as much control over your test data to achieve the tests you require. Stub-O-Matic utilizes the ubiquitous HTTP protocol to receive requests and return stub responses to the system under test. 

To use stubs, the system under test must be capable of diverting requests that would normally go to back-end systems. How this is best done depends on the architecture of the system under test. 

See Service Virtualization for further information.

Features

  • Test tool and methodology agnostic
  • Enterprise grade
  • Fast, thousands of concurrent stub requests per second are supported
  • Stubs can be hand crafted or recorded from real backends, or a combination of both
  • Everything in Stubo is scriptable and repeatable
  • Stubs can be mastered in a source control system along with tests and code
  • Stubbing can be managed programmatically through a RESTful API
  • Configurable delays can be added to simulate backend response times
  • Each test effectively owns its own test data
  • Stubbed responses can be templated selecting values from the request or system
  • Stateful scenarios can be simulated
  • Test stubs can be recorded from the backend service and reused
  • Dynamic request and response matching, e.g. to roll dates, ignoring IDs etc.
